Análisis y Diseño

  • Modelo de datos persistentes (Base de datos) -- Esta plantilla debe contener el diagrama respectivo al modelo de datos persistentes de la aplicación y la especificación de los datos a intercambiar con otras aplicaciones, en caso de requerirse el intercambio de datos, puede utilizar modelos de base de datos relacional o NOSQL

  • Arquitectura de software -- Esta plantilla debe contener los diagramas utilizados para representar la arquitectura del software, puede utilizar los diagramas del lenguaje UML para definir la arquitectura.

  • Prototipo no funcional de la interfaz de usuario -- Esta plantilla debe contener el prototipo no funcional de la interfaz de usuarios, puede utilizar storyboards, mockups, o herramientas como Pincel.
